Lesson 3 · 核心概念

Pi 的会话生命周期

时长约 7 分钟

上一课你学会了安装和启动 Pi。 现在来理解 Pi 的会话生命周期——从敲下 pi 到 exit 退出的完整流程。 理解了这个，你就能看懂 >> 提示符背后到底在发生什么。

本课目标 到这一课结束时，你应该能用自己的话描述：启动 Pi 后发生了什么、你按下回车后 Pi 如何思考并行动、以及一次完整的 "用户提问 → LLM 思考 → 执行工具 → 返回结果" 循环是怎么跑通的。

从敲下 pi 到 exit 退出

Pi 的完整生命周期分为三个阶段。每个阶段都有明确的事要做。 

启动
  │
  ├─ ① 加载扩展 → 扫描 .pi/extensions/ 和 ~/.pi/agent/extensions/
  │              → 扫描 npm 包
  │
  ├─ ② 初始化 Provider → 加载 API Key / OAuth 凭据
  │                   → 初始化 LLM 连接
  │
  └─ ③ 进入交互循环 → 等待你输入
                 → 按下回车，开始 Agent 循环
                 → exit，清理资源，回到 shell

阶段 1：加载扩展

Pi 启动时，第一步是扫描你（或第三方）写的 Extension。 

扫描顺序：
  1. 项目本地：.pi/extensions/*.ts
  2. 用户全局：~/.pi/agent/extensions/*.ts
  3. npm 包：pi install 安装的第三方包

扩展被加载后，它们用 pi.registerTool()、pi.registerCommand() 等方法向 Pi 注册能力。 扩展加载的顺序决定了能力的优先级。

为什么先加载扩展？ Agent 循环中 LLM 能调用的工具列表，是在扩展加载阶段就定好的。扩展没加载，工具就不存在。

阶段 2：初始化 Provider

扩展加载完后，Pi 初始化 LLM 连接。 

凭据来源（优先级从高到低）：
  1. --api-key 命令行参数
  2. ~/.pi/agent/auth.json（/login 写入）
  3. 环境变量（ANTHROPIC_API_KEY、OPENAI_API_KEY 等）

如果找不到凭据，Pi 会提示你去 /login。没有 LLM 连接，Agent 循环跑不起来。

阶段 3：交互循环——按下回车后

前两个阶段完成后，你看到 >> 提示符。 这是最值得理解的部分——你按下回车后，Pi 内部发生的事： 

你输入："列出项目里的所有 ts 文件"
          │
          ▼
  ┌─────────────────────────────────────┐
  │  Agent Loop（重复直到回答完成）     │
  │                                     │
  │  ① LLM 思考（Thinking）             │
  │     "用户要列出 ts 文件，           │
  │      我应该调用 find 工具"          │
  │                                     │
  │  ② LLM 发出工具调用（Tool Call）    │
  │     { name: "find",                  │
  │       arguments: { pattern: "*.ts" } }│
  │                                     │
  │  ③ Pi 执行工具（Execute）           │
  │     运行 shell 命令，返回结果        │
  │                                     │
  │  ④ LLM 观察结果（Observation）      │
  │     "找到了 23 个文件，             │
  │      我来整理一下回复用户"          │
  │                                     │
  │  ⑤ LLM 回复你（Response）           │
  │     "项目里有 23 个 .ts 文件..."     │
  └─────────────────────────────────────┘
          │
          ▼
  >>>  （等待下一次输入）

这个循环称为 Agent Loop——思考、行动、观察，直到任务完成。 大部分时候你看不到中间步骤，Pi 会自动完成工具调用然后给你一个综合回复。

四个界面区域

Pi 的界面固定分为四个区域，从上到下：

区域位置内容
顶部信息栏 最上方 快捷键、已加载的 AGENTS.md、当前模型
消息区 中间 你的消息、LLM 的回复、工具调用和结果
输入框 消息区下方 输入自然语言指令的地方，边框颜色表示思考等级
底部状态栏 最下方 工作目录、session 名、token 用量、当前模型

退出

exit 或 Ctrl+C 两次。 Pi 清理资源，保存 session 到 ~/.pi/agent/sessions/，然后回到 shell。

2. 你按下回车后，LLM 执行的"思考→行动→观察"循环叫做？
3. Session 退出时保存在哪里？
推荐阅读 Agent 循环的源码在 packages/agent-core/，交互模式实现在 packages/coding-agent/src/modes/interactive/。想深入可以读 agent-session.ts。
