Lesson 2 · 入门

安装 Pi，并完成你的第一次启动

时长约 5 分钟

上一课我们建立了心智模型。 现在要把 Pi 装到你的电脑上——不装，什么都做不了。

本课目标 到这一课结束时，你应该能独立完成：安装 → 启动 → 退出 Pi，并理解每个步骤在干什么。

安装

Pi 是一个 npm 包，安装只需要一条命令： 

npm install -g @earendil-works/pi-coding-agent

-g 表示全局安装——装完后你在任何目录都能直接敲 pi。

前提：你需要先装好 Node.js（推荐 18 或更高）。检查：node --version。

启动

安装完成后，在任何目录下运行： 

pi

这就是启动命令。第一次启动时，Pi 会初始化一些目录结构， 然后进入交互模式——你在终端里看到一个 >> 提示符， 可以在里面输入自然语言指令。 

$ pi
✓ loaded extension: hello.ts
pi>> 你好，你是谁？
扩展加载顺序 首次启动时 Pi 会扫描：.pi/extensions/（当前项目）→ ~/.pi/agent/extensions/（全局）→ npm 包。如果某个目录不存在，跳过，不报错。

退出

结束会话，回到 shell： 

exit

或者按 Ctrl+C 两次。和大多数 CLI 一样，没有什么特殊的退出快捷键。

完整流程一览

步骤命令说明
安装 npm install -g @earendil-works/pi-coding-agent 全局安装，一次就行
启动 pi 进入交互模式
退出 exit 或 Ctrl+C×2 回到 shell

练习

完成你的第一次 Pi 对话

  1. 确认安装：pi --version（输出版本号表示装好了）
  2. 启动：pi
  3. 输入一句话：现在几点了？
  4. 观察它的回复
  5. 退出：exit

这就是你人生第一个 Pi 会话。

检查你的理解

1. 安装 Pi 的正确命令是？
2. 启动 Pi 的命令是？
3. 退出 Pi 的正确方式是？
推荐阅读 Pi 官方 README：github.com/earendil-works/pi。读完介绍段落后直接看 Installation 部分，确认本地安装成功。
遇到问题？ pi 命令找不到？检查 npm install -g 的全局 bin 目录是否在 PATH 里：npm bin -g。把那个目录加到 PATH 即可。
