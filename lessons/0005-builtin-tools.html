Lesson 5 · 工具篇

四个内置工具详解

时长约 7 分钟

Pi 内置了四个基础工具：read、write、edit、bash。 它们是 Agent Loop 里 LLM 最常调用的操作。

本课目标 学完这一课，你应该能说出每个工具的参数、适用场景和限制，并理解 LLM 如何决定用哪个工具。

一图看懂

工具用途典型参数返回值
read 读取文件内容 path, offset, limit 文件内容（文本或图片）
bash 执行 shell 命令 command, timeout stdout/stderr、退出码
edit 精确替换文件内容 path, edits (旧字符串→新字符串) diff 预览
write 创建或覆盖文件 path, content 写入确认

另外还有三个辅助只读工具：grep、find、ls，它们默认也可用，但核心操作还是上面四个。

① read —— 读文件

最常用的工具。参数：

  • path（必须）：相对或绝对路径。
  • offset（可选）：从第几行开始读（1-indexed）。
  • limit（可选）：最多读多少行。
# 读整个文件
read(path="package.json")

# 读第 10-20 行
read(path="src/app.ts", offset=10, limit=11)

支持图片：粘贴图片或拖入终端时，Pi 会将其转为 base64 图像内容给 LLM。

返回值截断 大文件会被截断（默认 5000 行或 200KB），并在末尾显示截断提示，避免撑爆上下文。

② bash —— 运行 shell 命令

执行任意 shell 命令，返回输出。参数：

  • command（必须）：要执行的命令。
  • timeout（可选）：超时秒数，无默认值（命令可无限运行）。
bash(command="find . -name '*.ts'")
bash(command="npm run build", timeout=60)

注意：

  • 命令在 Pi 的工作目录（cwd）中执行。
  • 输出会被截断（默认 5000 行或 200KB），截断部分会在尾部标记。
  • 交互式命令（如 vim）不适合直接调用，因为 Pi 不是交互式 shell。
安全限制 Pi 会限制命令执行环境，但本质上它能在你的项目目录运行任何命令——谨慎对待你让 Pi 运行的命令。

③ edit —— 精确替换

edit 是 Pi 最强大的编辑工具，它通过精确字符串匹配来修改文件。

edit(
  path="src/index.ts",
  edits: [
    { oldText: "const port = 3000;", newText: "const port = 8080;" },
    { oldText: "console.log('start')", newText: "// console.log('start')" }
  ]
)

参数：

  • path：文件路径。
  • edits：一个数组，每个元素包含 oldText 和 newText。

关键规则：

  1. oldText 必须在文件中唯一出现——如果出现多次，编辑会失败。
  2. 多个 edits 不能重叠或嵌套——每个编辑针对原文件独立匹配。
  3. 匹配是精确的——包括空格、换行、缩进。LLM 必须从 read 复制原内容。
为什么不用行号？ 因为行号在代码变更中容易漂移，精确字符串匹配更鲁棒。LLM 通过 read 拿到精确内容，再交给 edit。

④ write —— 创建或覆盖文件

直接写入整个文件内容。参数：

  • path：文件路径。
  • content：要写入的完整内容。
write(
  path="newfile.txt",
  content="Hello, world!\n"
)

如果文件已存在，write 会覆盖它。通常 edit 更安全，因为只改小片段；write 用于创建新文件或完全重写。Pi 不会自动备份，所以建议在版本控制（git）下使用。

LLM 如何选择工具？

LLM 在 Agent Loop 中根据你的指令和上下文，自主决定调用哪个工具。通常的决策逻辑：

  • 需要查看信息 → read（或 grep/find/ls）
  • 需要运行命令 → bash
  • 需要修改文件 → 优先 edit（精确、安全），只有创建新文件或完全重写时才用 write

你不需要手动指定工具，只要清晰描述任务，LLM 会组合这些工具来完成任务。

练习

识别工具调用

下面是一个 Pi 会话片段。判断每个工具调用对应哪个内置工具：

  1. pi 执行了 { name: "read", arguments: { path: "src/main.rs" } }
  2. pi 执行了 { name: "bash", arguments: { command: "cargo build" } }
  3. pi 执行了 { name: "edit", arguments: { path: "Cargo.toml", edits: [{ oldText: "version = \"0.1.0\"", newText: "version = \"0.2.0\"" }] } }

答案很明显——但想想 LLM 为什么按这个顺序调用它们。

检查你的理解

1. 哪个工具用于精确修改文件中的小片段？
2. edit 要求 oldText 在文件中必须？
3. 哪个工具通常用于创建新文件？
推荐阅读 源码中每个工具的定义：packages/coding-agent/src/core/tools/。看 read.ts、bash.ts、edit.ts、write.ts 的 Schema 定义就能理解参数。
老师在线 "为什么编辑时 oldText 必须唯一？"、"bash 命令如果超时会怎样？"、"能同时用多个 edit 吗？"——随时问。
