注册 Slash 命令与拦截事件

时长约 8 分钟

第 9 课你学会了用 registerTool 让 LLM 调用你的函数。 但 Extension 的能力不止于此——你还可以注册 Slash 命令让用户主动触发功能， 以及拦截事件在 LLM 调用工具前后插入自定义逻辑。

本课目标 到这一课结束时，你应该能：用 pi.registerCommand 添加一个 /mycommand； 用 pi.on("tool_call", ...) 在工具执行前做校验或日志； 并理解两种方式各自的使用场景。

回顾：四种注册方式

ExtensionAPI 提供了四类注册方法，覆盖 LLM 和用户的交互入口：

方法影响对象触发方第 9 课学过
pi.registerTool LLM 的工具列表 LLM 自动调用 是
pi.registerCommand Slash 命令列表 用户输入 /xxx 否
pi.on(…) Agent 生命周期事件 事件触发时自动回调 否
pi.appendEntry Session 持久化 主动调用写入 否

registerTool 学会了，剩下三种在第 10–12 课逐个解锁。这一课学 registerCommand 和 pi.on。

pi.registerCommand——让用户主动触发

工具由 LLM 决定什么时候调用，但命令由用户决定什么时候执行。用户在对话中输入 /命令名，Pi 立即执行你注册的处理函数。

import type { ExtensionAPI } from "@earendil-works/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  pi.registerCommand({
    name: "status",
    description: "Show current session stats",
    execute: async (ctx) => {
      const msgs = await ctx.getMessageCount();
      return `当前对话有 ${msgs} 条消息。`;
    },
  });
}

保存后重启 Pi，输入 /status 即可看到输出。

命令 vs 工具 命令的 execute 只接收 ExecutionContext，没有参数对象——参数从命令文本中解析。 命令处理完直接输出文本，不经过 LLM。

命令参数

命令可以带参数，Pi 会把命令行的剩余部分作为字符串传入：

pi.registerCommand({
  name: "echo",
  description: "Repeat what you type",
  execute: async (ctx, args: string) => {
    if (!args.trim()) return "说点什么？";
    return `你说了: ${args}`;
  },
});

在 Pi 中输入 /echo 你好世界，返回 你说了: 你好世界。

动手：氛围切换器

把第 9 课的 random.ts 扩展一下，加一个 /roll 命令，让用户直接掷骰子而不用等 LLM：

import type { ExtensionAPI, ExecutionContext } from "@earendil-works/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  // 工具方式：LLM 调用
  pi.registerTool({
    name: "roll_dice",
    description: "Roll a dice, returning an integer between 1 and n",
    parameters: {
      type: "object",
      properties: {
        n: { type: "number", description: "Max value (>=1)" },
      },
      required: ["n"],
    },
    execute: async (_ctx, params) => {
      const n = Number(params.n);
      const val = Math.floor(Math.random() * n) + 1;
      return { content: [{ type: "text", text: `${val}` }] };
    },
  });

  // 命令方式：用户主动触发
  pi.registerCommand({
    name: "roll",
    description: "Roll a dice. Usage: /roll <max>",
    execute: async (_ctx, args: string) => {
      const n = parseInt(args.trim(), 10);
      if (isNaN(n) || n < 1) return "用法: /roll <最大数字>";
      const val = Math.floor(Math.random() * n) + 1;
      return `${val}`;
    },
  });
}

动手试试

  1. 把上面代码保存到 ~/.pi/agent/extensions/random.ts（覆盖旧文件）。
  2. 重启 Pi。
  3. 输入 /roll 20，确认直接返回数字。
  4. 再试试 帮我掷一个 6 面的骰子，确认 roll_dice 工具也正常。

pi.on()——拦截 Agent 事件

Extension 不仅可以注册新能力，还可以监听和拦截 Agent 的运行事件。 最常见的两个事件是 tool_call（工具即将执行）和 tool_result（工具已返回）。

事件名触发时机可以
tool_call LLM 请求调用工具时 校验参数、记录日志、修改参数、或阻止执行
tool_result 工具执行完成时 修改返回值、记录耗时
response LLM 生成回复时 在最终回复中加入额外信息

例子：记录所有工具调用

export default function (pi: ExtensionAPI) {
  pi.on("tool_call", (ctx, toolCall) => {
    ctx.log(`[audit] tool=${toolCall.name} args=${JSON.stringify(toolCall.args)}`);
  });
  pi.on("tool_result", (ctx, toolCall, result) => {
    ctx.log(`[audit] done=${toolCall.name} elapsed=${result.elapsed}ms`);
  });
}

拦截和阻止

在 tool_call 事件中，如果工具参数不合理，可以阻止它执行：

pi.on("tool_call", (ctx, toolCall) => {
  if (toolCall.name === "bash" && /rm\s+-rf/.test(toolCall.args.command)) {
    ctx.ui.confirm("检测到危险命令，确认执行？").then(ok => {
      if (!ok) throw new Error("用户取消了危险操作");
    });
  }
});
设计意图 事件钩子的价值不是"控制用户"——而是在工具调用流中插入审计、节流、权限检查等横切关注点。 比如：记录所有 write 调用的文件路径，或者阻止对 /etc 的写入。
