Lesson 14 · 加载与分发

扩展加载机制 & Pi Package 分发

时长约 8 分钟

前面 13 课你写了 Extension、注册了 Provider、编了 Skill。 现在搞清楚——这些文件到底从哪加载，优先级是什么，怎么把扩展分享给别人。

本课目标 到这一课结束时，你应该能：说出三种扩展加载路径的加载顺序和覆盖关系； 用 pi install 安装一个别人写好的扩展； 知道怎么把一个扩展打包成可共享的 Pi Package。

三种加载路径

Pi 在启动时会扫描以下目录中的扩展文件：

路径范围类型优先级
~/.pi/agent/extensions/ 全局——所有项目都生效 .ts / .js / .md 最高
.pi/extensions/ 项目本地——仅当前项目生效 .ts / .js 中
npm 包（pi install 安装） 全局/项目 打包的扩展包 最低

覆盖关系：同一名字的扩展，本地路径会覆盖全局路径，全局路径会覆盖 npm 包。 这就是为什么你可以用全局 Extension 来"劫持"某个内置功能或第三方扩展。

加载顺序 Pi 的加载顺序是：内置 → npm 包 → 项目本地 → 全局。 后加载的扩展会覆盖先加载的同名扩展。

扩展文件命名与发现

在 ~/.pi/agent/extensions/ 或 .pi/extensions/ 中，Pi 会：

  • 加载所有 *.ts 和 *.js 文件（自动转换 TypeScript）
  • 加载所有 *.md 文件作为 Skill（如果文件匹配 Skill 格式）
  • 跳过以 _ 或 . 开头的文件（隐藏文件或私用文件）
  • 如果扩展文件以 provider-* 开头，自动识别为 Provider Extension
  • 如果扩展文件名是 skills.md，会作为技能文档加载（与 SKILL.md 类似）

命名约定

虽然没有强制要求，但建议：

  • Provider 扩展：provider-deepseek.ts、provider-ollama.ts
  • 工具/命令扩展：feishu.ts、git-helper.ts
  • Skill：python-style.md、code-review.md

扩展文件结构

一个完整的 Extension 文件可以是：

// ~/.pi/agent/extensions/feishu.ts
import type { ExtensionAPI, ExecutionContext } from "@earendil-works/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  // 注册工具
  pi.registerTool({
    name: "feishu_send",
    description: "Send a message to Feishu group",
    parameters: {
      type: "object",
      properties: {
        message: { type: "string", description: "Message content" },
      },
      required: ["message"],
    },
    execute: async (_ctx, params: { message: string }) => {
      // 调用飞书 API 逻辑
      return { content: [{ type: "text", text: "Sent." }] };
    },
  });

  // 注册命令
  pi.registerCommand({
    name: "feishu-status",
    description: "Check Feishu bot status",
    execute: async (_ctx) => {
      return "Feishu bot is online.";
    },
  });
}

注意：扩展名就是它在 Pi 中的"身份标识"。 如果你需要更新扩展，只需要修改同名文件； 如果你想要"禁用"某个扩展，把文件重命名（比如加 .disabled 后缀）即可。

热重载 vs 重启 Extension 加载后，后续修改不会自动生效，需要退出 Pi 重新启动。 Skill 文件修改后，用 /reset 清空上下文即可生效（因为 Skill 是注入到系统提示词中的）。

用 pi install 安装扩展包

除了手写扩展文件，Pi 还支持通过 pi install 安装别人分享好的扩展包。 这些包以 npm 包的形式存在，通过 pi install <package-name> 安装。 

# 安装一个扩展包
pi install pi-skill-git-commit

# 安装后查看已安装的扩展
pi list

# 卸载
pi uninstall pi-skill-git-commit

安装后，Pi 会自动：

  • 在 ~/.pi/agent/ 下查找该包提供的扩展文件并加载
  • 检查是否有 Skill 文件（SKILL.md 或 *.md）并注入系统提示词
  • 如果发现是 Provider 扩展，注册到模型列表
包名约定 官方扩展包通常以 @pi/ 或 pi- 开头，但第三方包没有强制要求。 安装前建议先看一下包的内容，确保不包含恶意代码。

扩展包的目录结构

一个可共享的扩展包通常有以下结构： 

pi-skill-my-tool/
├── package.json
├── extensions/
│   └── my-tool.ts
├── skills/
│   └── my-tool.md
└── README.md

package.json 里需要声明扩展类型： 

{
  "name": "pi-skill-my-tool",
  "pi": {
    "type": "extension",
    "description": "My custom tool for X",
    "entry": "extensions/my-tool.ts"
  }
}
