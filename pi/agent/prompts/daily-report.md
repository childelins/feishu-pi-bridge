请帮我整理一份"今日 AI × 科技日报"。三个数据源聚焦不同领域，需要**分别调用对应的 skill** 拿实时数据，不要凭训练数据脑补。

## 步骤

1. **获取单天日期**：date +%Y-%m-%d。
2. **AI 前沿**：调用 **aihot** skill，对它说"**看看今天的ai日报**"。
3. **今日热点**：调用 **tavily-search** skill，对它说"**看看今天的热门新闻**"，注意：聚焦社会新闻（中国大陆)，至少 10 条。
4. **GitHub Trending**：调用 **github-trending** skill，对它说"**看看今天的热门仓库**"。

## 输出格式（固定排版，逐条遵守）

全程中文 markdown，按以下结构输出，不增删板块、不改标题层级：

日期：<当天日期，YYYY-MM-DD>

## 1) AI 前沿
> 数据来源：aihot skill ·「看看今天的ai日报」

按当天实际内容划分子分类（不限数量），每个子分类用 ### 三级标题，其下用有序列表（每个子分类内从 1 重新编号、连续不跳号），每条三行：
### <子分类，如：模型发布 / 更新、产品 / 工具更新、行业动态、论文 / 研究>
1. <标题>
   <一句话亮点或摘要，40 字以内>
   链接：<原文 URL>

本板块结尾固定：
### 今日一句话总结
<一句话概括今天 AI 圈主线>

## 2) 今日热点
> 数据来源：tavily-search skill ·「看看今天的热门新闻」，聚焦中国大陆社会新闻

### 今日较受关注的社会面热点
无序列表，每条：
- <标题>
  <一句话说明>
  参考：<URL，多个则逐行排>

至少 10 条。结尾固定：
### 今日热点一句话总结
<一句话>
（仅当该源噪音明显时，在总结后追加一行 > 说明：<简述取舍依据>；否则省略）

## 3) GitHub Trending
> 数据来源：github-trending skill ·「看看今天的热门仓库」

### 今日热门仓库
无序列表，每条四行：
- <owner/repo>
  <一句话简介>
  今日增长：<N> stars
  <URL>

尽量列全（不限数量）。结尾固定：
### GitHub 今日观察
<一句话趋势观察>

## 总结
- AI 前沿：<一句话>
- 社会热点：<一句话>
- GitHub Trending：<一句话>

硬性约束：
1. 不要写一级标题（#）；卡片标题由 bridge 自动生成，保留带日期的默认标题。
2. 列表符号统一：AI 前沿用有序列表且每个子分类内编号连续；今日热点、GitHub 用 - 无序列表；条目之间空一行。
3. 每条信息必须附原文链接；取不到链接的条目一律舍去，严禁凭训练数据编造链接或内容。
4. 禁止输出 tool/skill 的调用过程、重试、异常等中间信息（如"现在我已收集到所有三个数据源的内容，开始整理报告", "我现在开始并行获取三个数据源。先确认需要的工具可用，然后同时拉取数据。", "三个数据源 agent 已并行启动。等待它们返回结果"）。
5. 报告在「总结」最后一条之后立即结束，禁止任何对话式收尾（如"如果你愿意我可以…"）。
6. 不要通过临时文件写入到本地环境，不要会导致无法给到相关内容。
