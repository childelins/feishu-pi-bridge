Lesson 7 · 进阶

Session 管理与分支

时长约 6 分钟

Pi 自动保存你的每次对话为 Session。 理解 Session 怎么工作，能让你在复杂任务中保持方向感，并在需要时回到任意节点继续。

本课目标 到这一课结束时，你应该能：用 /resume 恢复之前的对话、用 /tree 在历史里跳转和分支、用 /fork 和 /clone 创建新分支，并在会话变长时用 /compact 手动压缩。

Session 是什么？

Session 就是 Pi 自动保存的对话记录。每次你启动 pi，它创建一个 Session； 你退出时，Session 被保存到磁盘。 

~/.pi/agent/sessions/
├── --home-project--/
│   ├── 2024-07-07_abc123.jsonl
│   └── 2024-07-07_def456.jsonl
└── --work-project--/
    └── 2024-07-07_ghi789.jsonl

每个文件是一个 Session，内容按时间戳命名，按项目目录分组。

恢复 Session

继续最近的对话，用 pi -c 或在交互模式里输入 /resume。

# 启动时继续最近的 session
pi -c

# 启动时浏览并选择一个 session
pi -r

# 在交互模式里打开 session 选择器
>> /resume

选择器支持搜索（直接打字）、切换路径显示（Ctrl+P）、按名称过滤（Ctrl+N）。

命名 Session：用 /name Refactor auth module 给当前 Session 起名。命名后的 Session 在选择器里更容易找到。

Session 是树，不是线

Pi 的 Session 不是简单的线性记录——它是一个树结构。 每个节点都有 id 和 parentId，从根到叶子的路径构成一个「活跃分支」。 

        [user: "怎么重构这个函数？"]
              │
        [assistant: "建议抽成三个函数"]
        /                  \
[user: "好，方案 A"]     [user: "方案 B 呢？"]
        │                       │
  [assistant: ...]         [assistant: ...]
        │                       │
   [active leaf]           [另一个分支]

这种设计意味着：你可以随时跳到之前的任意位置，从那里创建新分支继续，而不会丢失其他分支的上下文。

/tree — 在历史里跳转

/tree 让你在当前 Session 的树里自由导航。

在 Pi 交互模式里输入 /tree

进入树视图后：

快捷键做什么
↑ / ↓上下移动
Enter选择并继续
Escape取消
Shift+L给节点打标签（书签）
Ctrl+O切换过滤模式

/tree 不创建新文件——它在当前 Session 内跳转。适合探索不同尝试方向，所有历史保留在同一文件里。

/fork — 创建新 Session 分支

当你确定要从某个历史点彻底分叉，用 /fork： 创建一个新 Session 文件，从选中的节点开始继续。 

在 /tree 里选好要分叉的节点 → /fork → 新 Session 文件创建完成

/fork vs /tree：/tree 在原文件里跳转；/fork 创建新文件，保留两个独立分支。用 /fork 当你确定分叉是一个独立方向。

/clone — 复制当前分支

/clone 是 /fork 的简化版——直接复制当前活跃分支，从当前位置开始。

>> /clone  → 当前分支的完整历史被复制，新 session 打开为空编辑器

/compact — 压缩上下文

Session 太长时，上下文会逼近模型的 token 上限。此时 Pi 会自动触发 Compaction（压缩）—— 把旧消息总结成摘要，保留摘要 + 最近的完整消息。

手动触发：

>> /compact 或 >> /compact 聚焦于 bug 修复相关上下文

可选的 prompt 让你控制压缩的方向——比如"只保留 API 设计和当前 bug 相关的上下文"。

一个完整的分支工作流

在 Pi 里练习分支

  1. 启动 Pi，发送消息："我们来做一个 3 层塔的算法"
  2. 等它回复后，发送消息："不，换个思路，试试动态规划"
  3. 按 Escape 两次（或 /tree），切回到它第一个回复，按 Enter
  4. 在分支 A 上继续："继续，递归解法怎么写？"
  5. 退出后重新启动 Pi，/resume 找到刚才的 session，观察两个分支

如果今天只练一个，就练 /tree 切回分支——这是最有用的能力。
