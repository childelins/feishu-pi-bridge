Lesson 6 · 使用进阶

Slash 命令与 Context Files

时长约 6 分钟

前五课你掌握了 Pi 的核心模型。现在学最常用的两个日常操作： Slash 命令是 Pi 里的「快捷键」，Context Files是给 Pi 下指令的标准方式。

本课目标 到这一课结束时，你应该知道 6 个最常用的 Slash 命令、Context Files 放哪里怎么写， 并能在日常对话中自然使用它们。

Slash 命令是什么？

在输入框里输入 / 开头，Pi 就会把它当成一条命令而不是普通对话。 

>> /model
>> /reload
>> /session

Pi 有 18+ 个 Slash 命令，但 你日常只需要记住 6 个，其余知道存在就行。

最常用的 6 个 Slash 命令

命令做什么什么时候用
/model 切换模型（等价于 Ctrl+L） 当前模型太慢或不够聪明时
/settings 修改 thinking 等级、主题、消息投递等 想调高 thinking 等级、换主题时
/resume 浏览和恢复之前的 session 想回到之前的工作上下文
/new 开始一个新 session（清空上下文） 切任务了，不想让历史影响新对话
/reload 重新加载 Extension、Settings、Context Files 改了设置或文件后不想重启 Pi
/quit 退出 Pi（等价于 exit） 不解释了
2 个 Session 深度命令 /tree — 在会话历史里跳转和分支，同一会话文件内切到之前的任意位置继续。 /compact — 手动压缩上下文，把旧消息总结成摘要以腾出空间。 这两个稍复杂，但用得多了会知道它们的存在。

输入技巧

输入框里还能做这些：

操作效果示例
@文件名 模糊搜索文件并附加到消息中 @package.json 这个依赖了哪些包？
!命令 执行 shell 命令，输出发给 LLM !npm run build
!!命令 执行 shell 命令，输出不发给 LLM !!pkill -f dev-server
Shift+Tab 切换 thinking 等级 从 medium 切到 high
Ctrl+O 展开/折叠工具调用输出 想看 read 返回了什么内容

Context Files（AGENTS.md）

Context Files 是 Pi 启动时自动加载的 Markdown 文件，用于告诉 Pi 一个项目的规则和约定。它们不是工具，而是注入到系统提示词里的背景知识。

存放位置

Pi 启动时会按以下顺序查找并加载所有匹配的文件：

1. ~/.pi/agent/AGENTS.md         # 全局指令（所有项目生效）
2. 从 cwd 往上查找 AGENTS.md     # 项目目录及其父目录
3. cwd 下的 CLAUDE.md             # 兼容 Claude Code 的配置文件

所有找到的文件内容会合并在一起，追加到系统提示词中。

写什么

# 项目规则

- 运行 `npm run check` 通过后才能提交
- 不要修改 `dist/` 目录下的文件
- 使用 2 个空格的缩进
- 测试框架使用 Vitest
修改后要 /reload 修改了 AGENTS.md 后，Pi 不会自动重新加载。运行 /reload 让它重新读取，或者重启 Pi。

扩展模式也能注册命令

你现在理解 Slash 命令是 Pi 内置的交互方式，但这为后面学习 Extension 埋了一个关联： Extension 可以通过 pi.registerCommand() 添加自定义 Slash 命令。 你后面写飞书 Extension 时，可以用它注册一个 /send-to-feishu 命令。

练习

写一个项目级的 AGENTS.md

  1. 在项目根目录创建 AGENTS.md
  2. 写上 3 条你希望 Pi 遵守的规则
  3. 启动 Pi（或 /reload），发一条消息看看它是否遵守
  4. 再试试 /model 切换一下模型

如果你已经建立了 AGENTS.md，尝试修改它，用 /reload 重新加载，再对比修改前后的表现。
