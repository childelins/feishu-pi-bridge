Lesson 8 · 概念 + 模式

Extension 是什么 & Pi 的四种运行模式

时长约 7 分钟

进入写 Extension 之前，先把概念理清： Extension 是 Pi 的“插件系统”——在理解它之前，先搞懂它和 Skill、Prompt Template 有什么区别； 然后补上四种运行模式，因为 Extension 在不同模式下表现不同。

本课目标 到这一课结束时，你应该能说清楚 Extension 是什么、什么时候该写它， 以及 Pi 的四种运行模式分别适合什么场景。

Extension 是什么？

Extension 是一个 TypeScript 文件（或文件夹），用 export default 导出一个函数， 接收 ExtensionAPI 对象，然后通过 pi.registerTool()、pi.registerCommand() 等方法注册新能力。 

// 一个最简单的 Extension —— 注册一个工具
import type { ExtensionAPI } from "@earendil-works/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  pi.registerTool({
    name: "say_hello",
    description: "Say hello to someone",
    parameters: { type: "object", properties: { name: { type: "string" } } },
    execute: async ({ name }) => ({ content: [{ type: "text", text: `Hello, ${name}!` }] })
  });
}

把它放到 .pi/extensions/hello.ts，重启 Pi 后 LLM 就能调用 say_hello 工具。

存放位置 ~/.pi/agent/extensions/ 全局生效，.pi/extensions/ 项目本地生效。 用 pi -e ./my-ext.ts 临时加载测试。

Extension vs Skill vs Prompt Template

三者常常让人混淆，快速对比：

类型本质什么时候用
Extension TypeScript 代码，注册工具/命令/事件监听 需要程序化能力：读写文件、调 API、交互式 UI
Skill Markdown 文档，注入到系统提示词 教 LLM 做一件事的“配方”——重复可用的指令模板
Prompt Template 带变量的文本模板，展开为完整提示词 快速复用提问结构，不用写代码

简单判断：需要 执行代码 就用 Extension；需要 教 LLM 怎么做 就用 Skill；需要 快速复用提问模板 就用 Prompt Template。

Extension 能做什么？

  • 注册工具：让 LLM 调用你写的函数（pi.registerTool）
  • 注册 Slash 命令：添加 /mycommand（pi.registerCommand）
  • 拦截事件：在工具执行前拦截、修改或阻止（pi.on("tool_call", ...)）
  • 用户交互：弹出确认框、输入框（ctx.ui.confirm() / ctx.ui.input()）
  • 持久化状态：保存扩展自己的数据到 session（pi.appendEntry()）

四种运行模式

Extension 在不同运行模式下表现不同——比如 ctx.ui 在非交互模式下不可用。

模式命令适合场景Extension 可用性
交互模式 pi 日常对话、开发调试 全部可用（包括 UI）
一次性模式 pi -p "..." 脚本、CI 集成、单次提问 工具可用，UI 不可用
JSON 模式 pi --mode json 程序化消费完整事件流 工具可用，UI 不可用
RPC 模式 pi --mode rpc 让其它程序控制 Pi 工具可用，UI 不可用

判断规则：只要模式里带 --mode 或 -p，就说明 Pi 不会显示 UI，扩展的 ctx.ui 方法调用会失败或静默忽略。

写 Extension 时的模式意识：如果 Extension 需要用户交互，它只在交互模式下有效。 在非交互模式下，要么优雅降级，要么用日志替代弹窗。 下一课我们会看到怎么检查运行模式。
