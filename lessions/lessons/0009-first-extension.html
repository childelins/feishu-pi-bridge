Lesson 9 · 动手写扩展

写一个能跑的 Extension

时长约 8 分钟

第 8 课把 Extension 的概念讲清了，这一课动手写一个： 注册一个工具，让 LLM 能在对话中调用你写的函数。

本课目标 在写代码之前，先搞清楚 Extension 的基本结构：ExtensionAPI 接口、pi.registerTool 的签名、 以及 execute 回调的参数和返回值。然后亲手写一个"随机数生成器"，从建文件到在 Pi 里看到工具生效。

Extension 的骨架

每个 Extension 都是一个 TypeScript 文件导出一个函数，这个函数接收一个 ExtensionAPI 对象， 通过它注册工具、命令、事件监听等。 

import type { ExtensionAPI, Tool, ExecutionContext } from "@earendil-works/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  pi.registerTool({
    name: "my_tool",
    description: "Description for the LLM",
    parameters: { /* JSON Schema 定义参数 */ },
    execute: async (ctx: ExecutionContext, params: any) => {
      return {
        content: [{ type: "text", text: "返回给 LLM 的文本" }],
      };
    },
  });
}

四个必填字段，逐个说。

name

工具的唯一标识符。LLM 在生成 tool_call 时用它来引用你的函数。 只用小写字母、数字、下划线，避免空格和特殊字符。

description

给 LLM 看的。写清楚这个工具做什么、什么时候该用、什么时候不该用。 一句话说不完的，拆成短句，LLM 读得越清楚，调用的越准确。

parameters

标准的 JSON Schema 对象。定义函数接收的参数。 LLM 会根据 schema 生成调用参数，所以 properties 和 required 写准确很重要。

execute

你的代码逻辑在这里执行。参数是 ExecutionContext（提供 ctx.ui、ctx.log 等）， 第二个参数是 LLM 传过来的参数对象。 必须返回 { content: [{ type: "text", text: "..." }] } 格式， 返回的文本会直接喂给 LLM 作为工具调用的结果。

关于 parameters 如果不接受任何参数，传 { type: "object", properties: {} }。 每个 property 都可以指定 type（string/number/boolean）、description、 enum（可选值）、default 等。

动手：随机数生成器

建一个文件 ~/.pi/agent/extensions/random.ts，写一个 roll_dice 工具—— 给一个最大数 n，返回一个 1 到 n 的随机整数。 

import type { ExtensionAPI, ExecutionContext } from "@earendil-works/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  pi.registerTool({
    name: "roll_dice",
    description:
      "Roll a dice and return a random integer between 1 and n (inclusive). " +
      "Useful when you need a random number.",
    parameters: {
      type: "object",
      properties: {
        n: {
          type: "number",
          description: "Maximum value (inclusive). Must be at least 1.",
        },
      },
      required: ["n"],
    },
    execute: async (_ctx: ExecutionContext, params: { n: number }) => {
      const min = 1;
      const max = Math.max(min, Math.floor(params.n));
      const result = Math.floor(Math.random() * (max - min + 1)) + min;
      return {
        content: [{ type: "text", text: `Rolled: ${result} (range: ${min}-${max})` }],
      };
    },
  });
}

保存后，打开 Pi：pi，然后试试：帮我掷一个 20 面的骰子。 Pi 应该调用你的 roll_dice 工具并返回结果。

怎么知道扩展被加载了？ 启动 Pi 时，它会在终端输出加载了哪些扩展： Loaded 3 extensions: 之类的一行。 如果没看到你的扩展，检查文件路径和 TypeScript 语法。 用 pi -e ./random.ts 临时加载单个文件调试。

测试你的 Extension

动手试试

  1. 把上面的代码保存到 ~/.pi/agent/extensions/random.ts。
  2. 启动 Pi：运行 pi。
  3. 输入 掷一个 20 面的骰子，看 Pi 是否调用 roll_dice。
  4. 试试 掷一个 100 面的骰子，确认范围正确。
  5. 修改代码——把返回格式改成 🎲 ${result}，重启 Pi 再试一次。
热重载 Pi 不会自动检测 Extension 的变化。修改代码后需要退出再启动，或者用 /reset 清空当前会话后重新加载。

常见的第一个坑

parameters 和 execute 的类型要对齐。 如果 schema 里 n 是 number，但 execute 里把 params.n 当字符串用， LLM 传入的可能是 "20"（字符串）而不是 20（数字），会导致 NaN。

写一个防御性的小习惯：在 execute 里做类型检查或转换。 

const n = Number(params.n);
if (!Number.isInteger(n) || n < 1) {
  return `Error: n must be a positive integer, got ${params.n}`;
}
