Lesson 1 · Pi 入门

Pi 是什么，以及为什么它是一个值得你花时间的工具

时长约 6 分钟

在你写任何代码之前，先搞清楚 Pi 到底在解决什么问题。 这个问题的答案，决定了你后面每一行 Extension 代码该往哪里放。

这门课的目标 在这一课结束时，你应该能用自己的话回答：Pi 是什么、它和其他 AI 编程工具有什么不同、以及你为什么需要它。

一句话定义

Pi 是一个 终端里的 AI 编程助手。它把语言模型的能力直接嵌入到你的命令行工作流里，让你用自然语言和 shell 对话来写代码、改文件、跑命令。

但和 Claude Code、Cursor、GitHub Copilot 这些产品相比，Pi 的设计哲学非常不同——这一点是你后面组装它时要时刻记住的。

两种截然不同的产品哲学

大多数 AI 编程工具走的是 "all-in-one" 路线： 内置一切功能，开箱即用，你只需要用就行。

Pi 走的是 "minimal core + extensible" 路线：

All-in-One（如 Cursor）

内置功能
├── 补全
├── Chat
├── 终端 Agent
├── 数据库插件
├── 云端同步
└── 内置提示词
   ← 你选择使用
      ← 你无法替换

功能丰富，但定制度受限。

Minimal Core（如 Pi）

极简内核
├── read
├── write
├── edit
└── bash
   ← 这些是固定的
      ← 其它都靠你注入

四件工具起步，其它一切靠扩展。你可以塑形 Pi，而不是适应它。

Pi 作者 Mario Zechner 在介绍中说过："We build the minimum viable product and let you extend it."（我们做最精简的内核，由你来扩展它。）

这就是为什么你的 Mission 里写的是 "组装" 一个 Coding Agent—— 你不是在使用一个完成品，而是在用乐高积木搭一个属于你自己的东西。

Pi 现在能做什么？

不装任何扩展，Pi 开箱就能做这些：

  • 读文件——`read(path)`，读取任意文件内容
  • 写文件——`write(path, content)`，创建或覆盖文件
  • 改文件——`edit(path, old, new)`，精准替换某段内容
  • 跑命令——`bash(command)`，在 shell 里执行任意命令

这就是 全部。没有内置的 Git 工具，没有内置的搜索，没有内置的飞书通知，没有内置的数据库查询。一切都需要你通过扩展来添加。

设计上的取舍 这种极简不是偷懒，而是一种刻意。少意味着你知道每一个能力来自哪里，少意味着你可以完全控制它的行为，少意味着它不会在你不想的时候偷偷做什么。

你接下来要做什么？

你的目标很明确，拆成三个具体的东西：

  1. Extension——写一个 TypeScript 模块，接入飞书 API，让 Pi 能在飞书里发通知。
  2. 自定义 Provider——把私有 LLM 网关接进来，切换不同模型做不同任务。
  3. 自定义 Skill——把常用的业务审查流程封装成 Skill，让 Pi 自动学会。

这三件事本质上都是同一件事的不同侧面：给 Pi 注册新的能力。学会了其中一种，另外两种就水到渠成。

第一课不急着碰代码。先建立心智模型就够了。接下来的课我们会一步一步写。
