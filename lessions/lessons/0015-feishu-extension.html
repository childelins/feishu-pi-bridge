飞书长连接接入：不需要公网 IP

时长约 12 分钟

第 1–14 课，Extension 四种注册方式已全部解锁。 现在把它们串起来——用飞书长连接模式接入飞书，无需公网 IP。

MISSION 时刻 这一课直指 MISSION 核心目标："编写一个 Extension 来接入飞书 API"。 读完这一课，你应该能独立完成一个最小可用的飞书接入方案。

为什么用长连接，不用 Webhook

飞书提供两种事件接收方式，差别巨大：

请求网址（Webhook）长连接模式
需要公网 IP 是，要有可公网访问的 URL 否，本地直接运行
内网穿透 需要（ngrok / frp） 不需要
事件加密 是，要处理 Encrypt Key 否，明文推送
开发速度 约 1 周 约 5 分钟
消息处理超时 30 秒 3 秒

长连接模式的核心优势：你只需要能访问互联网，不需要公网 IP，不需要域名，不需要防火墙配置。直接在本地开发机上跑就行。

当前限制 长连接模式目前仅支持事件订阅，不支持回调订阅。 同时是集群模式：同一应用多客户端部署时，只有一个随机客户端会收到消息，不是广播。 3 秒内必须完成处理，否则触发超时重推。

飞书开放平台配置

  1. 在飞书开放平台创建企业自建应用
  2. 获取 App ID 和 App Secret
  3. 开启 机器人能力
  4. 在 事件订阅 里选择 长连接模式（不是请求网址）

  6. 发布应用到企业，把机器人拉进群

官方 SDK：@larksuiteoapi/node-sdk

飞书官方提供了完整的 Node.js SDK，包含 WebSocket 长连接客户端：

npm install @larksuiteoapi/node-sdk

完整代码：feishu-longconn.ts

新建 ~/.pi/agent/extensions/feishu-longconn.ts：

import * as Lark from "@larksuiteoapi/node-sdk";
import type { ExtensionAPI } from "@earendil-works/pi-coding-agent";

// ---- 飞书基础配置 ----

const baseConfig = {
  appId: process.env.FEISHU_APP_ID ?? "",
  appSecret: process.env.FEISHU_APP_SECRET ?? "",
};

// 飞书 REST API 客户端（用于发送消息）
const feishuClient = new Lark.Client(baseConfig);

// 飞书 WebSocket 长连接客户端（用于接收消息）
const feishuWS = new Lark.WSClient({
  ...baseConfig,
  loggerLevel: Lark.LoggerLevel.info,
});

// ---- 消息发送辅助函数 ----

async function replyText(chatId: string, text: string) {
  await feishuClient.im.v1.message.create({
    params: { receive_id_type: "chat_id" },
    data: {
      receive_id: chatId,
      content: JSON.stringify({ text }),
      msg_type: "text",
    },
  });
}

// ---- 启动长连接并接收消息 ----

feishuWS.start({
  eventDispatcher: new Lark.EventDispatcher({}).register({
    "im.message.receive_v1": async (data) => {
      const { message } = data as {
        message: { chat_id: string; content: string };
      };

      // 提取用户发送的文本
      const content = JSON.parse(message.content);
      const userText = content.text ?? "";

      if (!userText.trim()) return;

      // 获取 chat_id
      const chatId = message.chat_id;

      // 3 秒内必须完成，先立即回复"正在思考..."
      // 实际项目中可以先发一个"正在思考"消息
      await replyText(chatId, `收到：${userText}`);

      // TODO: 这里对接 Pi RPC
      // 把 userText 发送给 Pi，等待 Pi 的回复
      // 然后把 Pi 的回复用 replyText 发回飞书
    },
  }),
});

// ---- Pi RPC 接入点（下一节详解） ----

export default function (pi: ExtensionAPI) {
  // 这里注册 Pi 工具/命令/Provider
  // Pi 启动时也会加载 ~/.pi/agent/extensions/ 下的所有扩展
  // 包括你之前的 provider-deepseek.ts、feishu.ts 等
}
环境变量 在 shell 里设置：export FEISHU_APP_ID=cli_xxx 和 export FEISHU_APP_SECRET=xxx。 实际部署时建议用 dotenv 或系统环境变量管理。

和 Pi RPC 打通

飞书长连接负责接收消息，Pi RPC 负责处理消息。 把两者用管道串起来，就是完整的数据流。 

import * as Lark from "@larksuiteoapi/node-sdk";
import { spawn } from "child_process";

// ... feishu 配置同上 ...

// 启动 Pi RPC 进程
const piProcess = spawn("pi", ["--mode", "rpc", "--no-session"]);

// 缓冲 JSONL 行
let buf = "";
const pendingResolves = new Map<number, (v: any) => void>();
let nextId = 1;

piProcess.stdout.on("data", (chunk) => {
  buf += chunk.toString();

  while (true) {
    const nl = buf.indexOf("\n");
    if (nl === -1) break;

    let line = buf.slice(0, nl);
    buf = buf.slice(nl + 1);
    if (line.endsWith("\r")) line = line.slice(0, -1);

    try {
      const obj = JSON.parse(line);

      // 匹配 response（Pi 的命令回调）
      if (obj.type === "response" && obj.id != null) {
        const resolve = pendingResolves.get(obj.id);
        if (resolve) { resolve(obj); pendingResolves.delete(obj.id); }
      }

      // 匹配 agent_end（Pi 完成处理）
      if (obj.type === "agent_end") {
        const text = extractAssistantText(obj.messages ?? []);
        if (text) {
          replyText(lastChatId, text);
        }
      }
    } catch {}
  }
});

let lastChatId = "";

// 发送命令给 Pi
async function piPrompt(message: string) {
  return new Promise<void>((resolve) => {
    const id = nextId++;
    pendingResolves.set(id, () => resolve());
    piProcess.stdin.write(JSON.stringify({ id, type: "prompt", message }) + "\n");
  });
}

// 从 agent_end.messages 提取助手文本
function extractAssistantText(messages: any[]): string {
  return messages
    .filter((m) => m.role === "assistant")
    .map((m) => {
      if (Array.isArray(m.content)) {
        return m.content
          .filter((c) => c.type === "text")
          .map((c) => c.text)
          .join("\n");
      }
      return typeof m.content === "string" ? m.content : "";
    })
    .join("\n\n");
}

// 启动长连接
feishuWS.start({
  eventDispatcher: new Lark.EventDispatcher({}).register({
    "im.message.receive_v1": async (data) => {
      const { message } = data as { message: { chat_id: string; content: string } };
      const content = JSON.parse(message.content);
      const userText = content.text ?? "";

      if (!userText.trim()) return;
      lastChatId = message.chat_id;

      // 发给 Pi RPC
      await piPrompt(userText);
      // 回复会在 agent_end 里自动发回飞书
    },
  }),
});

数据流：飞书消息 → im.message.receive_v1 事件 → piPrompt() → Pi RPC prompt 命令 → agent_end 事件 → replyText() → 飞书消息 API

关键点拆解

飞书事件里有什么

im.message.receive_v1 事件的 data 结构：

{
  "message": {
    "chat_id": "oc_abc123",       // 群聊 ID
    "message_id": "om_abc123",     // 消息 ID
    "message_type": "text",
    "content": "{\"text\":\"Hello\"}",  // 消息内容（JSON 字符串）
    "sender": {
      "sender_id": { "open_id": "ou_abc123" }
    }
  }
}

message.content 是 JSON 字符串，需要先 JSON.parse()。

Pi RPC 的事件流

Pi RPC 模式通过 stdout 持续输出事件，关键事件：

事件说明
agent_start Pi 开始处理一条消息
message_update 流式更新，包含 text_delta（文本片段）
agent_end Pi 处理完成，包含 LLM 的所有回复消息
response 命令回调（如 prompt 命令被接受）

3 秒超时怎么办

长连接模式下飞书要求 3 秒内 完成处理。 Pi 处理复杂任务往往超过 3 秒。

两种策略：

  1. 先发"收到"→ 异步处理：收到消息后立即返回，用后台任务处理 Pi RPC，完成后再发一条新消息
  2. 超时设置 + 放弃：简单任务 3 秒内完成，复杂任务返回"任务已接收，请稍后查看"
// 策略 1 的伪代码
"im.message.receive_v1": async (data) => {
  const chatId = data.message.chat_id;
  const text = JSON.parse(data.message.content).text;

  // 立即确认收到（3 秒内完成）
  await replyText(chatId, "收到，正在处理…");

  // 异步交给 Pi
  await piPrompt(text); // 在 agent_end 里发最终回复
}
关于 Pi Extension 和飞书客户端的关系 上面的 feishu-longconn.ts 是 Pi 的 Extension，在 Pi 启动时加载。 但飞书长连接的 WSClient.start() 是个独立进程，不是 Pi 的一部分。 实际部署时，你可能有：
  • pi --mode rpc 进程
  • node feishu-longconn.ts 进程（或另一个脚本）
  • 或者把飞书客户端直接放在 Pi 的 Extension 里，通过 pi.on("message") 等机制对接
本课把它们合并到一个 Extension 文件里演示，方便理解全貌。
